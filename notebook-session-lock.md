# 计时保存 + 第二登录只读 方案

---

## 一、刷新/关闭页面时计时保存

### 现状
计时数据只存在内存里，页面一关就丢了。之前试的方案都因为清数据操作或云端覆盖而失败。

### 做法

1. 计时开始时，往 localStorage 写一个独立的备份记录，存下任务 ID 和开始时间
2. 页面关闭或刷新时，直接把已过秒数写入该任务的计时日志里，然后清理备份记录
3. 重新打开页面或登录时：
   - 先检查备份记录是否存在
   - 如果存在说明上次没正常关闭，计算差值，**叠加上去**（不替换已有数据）
   - 处理完清理备份
4. 备份记录是独立的存储 key，清空数据操作不碰它。恢复动作放在拉取云端数据**之后**执行，保证不会被云端旧数据覆盖。

## 二、第二个登录变为只读

### 现状
同一账号在两台设备或两个标签页同时开着，两个窗口都能编辑，数据互相踩。

### 做法

**同设备检测**：

1. 登录成功后在 localStorage 写一个心跳记录，存会话标识和当前时间
2. 每隔几秒刷新一次时间戳
3. 第二个标签页同账号登录时检测到这个心跳还在有效期内，自动进入只读模式
4. 主窗口关闭后心跳超时自动过期，另一个窗口可以接手
5. 退出登录时主动清理心跳

**跨设备检测**不强制只读，靠推送前先拉取再合并来缓解冲突。

**只读状态下**：

顶部显示红色横幅说明当前是只读模式。

不可操作：添加/编辑/删除事项、修改标签、开始/暂停计时、删除标签、导入导出、导入导出预设、手动设置时长。

仍可操作：日期前后翻页、回到今天、点击日期打开日历选择器、本日/本周/本月/累计切换、类型筛选和状态筛选、饼图图例点击展开收起、登录退出按钮。

用浏览器原生的 disabled 属性禁用输入框和按钮，用 pointer-events 禁用勾选框等自定义交互控件，安全列表里的元素显式恢复可点击。退出只读时全部恢复。
